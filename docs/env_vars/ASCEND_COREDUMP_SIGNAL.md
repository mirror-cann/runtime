# ASCEND\_COREDUMP\_SIGNAL

## 功能描述

设置trace处理的core dump信号量。

取值如下：

- none：关闭表1中的信号集。
- 非none的其他值：打开表1中的信号集。

trace默认开启堆栈功能，进程触发以下异常场景（信号）时，将栈信息记录在stackcore\_tracer\_\*.txt文件中。

**表1**  信号集

| 信号 | 信号值 |
| --- | --- |
| SIGINT | 2 |
| SIGQUIT | 3 |
| SIGILL | 4 |
| SIGTRAP | 5 |
| SIGABRT | 6 |
| SIGBUS | 7 |
| SIGFPE | 8 |
| SIGSEGV | 11 |
| SIGTERM | 15 |
| SIGXCPU | 24 |
| SIGXFSZ | 25 |
| SIGSYS | 31 |
| SIGRTMIN+1 | 35 |

若业务部署时，使用到了与表1中的信号注册有冲突的功能（比如JVM信号捕获，如下所示）时，需要通过该环境变量关闭信号注册。

关闭信号注册后，如果业务发生core dump异常，trace不会生成栈信息文件，需要借助其他手段定位异常。

## 配置示例

```sh
export ASCEND_COREDUMP_SIGNAL=none
```
